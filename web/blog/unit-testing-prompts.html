Unit Testing Prompts: Moving Beyond "Vibes"

Software engineering has TDD (Test Driven Development). Prompt engineering usually has "I looked at it and it seems okay." This must change.

Defining Success Metrics

You cannot improve what you cannot measure. A prompt like "Write a funny email" is untestable. A prompt like "Write an email containing fewer than 50 words and using one pun" is testable.

Assertions for LLMs

  • Length Check: `len(output) < 200`
  • Format Check: `is_valid_json(output)`
  • Keyword Check: `"unsubscribe" in output`
  • Semantic Check: Use a cheaper LLM (like GPT-3.5) to grade the output of a stronger LLM.

The Golden Set

Maintain a dataset of 50 "Golden Inputs" and their expected outputs. Every time you change your prompt, run it against the Golden Set to check for regression.
