Zero-Shot vs. Few-Shot: The ROI of Examples

Providing examples is the single highest-ROI activity in prompt engineering. Here is why.

Definitions

  • Zero-Shot: Asking the model to do something without examples. ("Classify this tweet.")
  • Few-Shot: Providing 3-5 examples of input -> output pairs. ("Tweet: 'I love this!' -> Positive. Tweet: 'This sucks.' -> Negative. Classify this: 'It's okay.'")

The Performance Gap

On complex reasoning tasks (like converting natural language to SQL), Few-Shot prompting can improve accuracy from 20% to over 80%. It establishes a pattern for the model to follow, drastically reducing the search space.

When to use which?

Use Zero-Shot for creative writing, brainstorming, or simple summaries.

Use Few-Shot for classification, code generation, data extraction, or anything requiring a rigid format.
